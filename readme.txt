Professional URL Conflict and Content Cannibalization Detector for WordPress.

== Description ==

SEO Cannibalization Scout is a powerful tool designed to help website owners identify and resolve content cannibalization issues. Content cannibalization occurs when multiple pages on your site target the same keyword, diluting your SEO authority and confusing search engines.

This plugin scans your published content, analyzes slugs and keywords, and identifies potential conflicts where URLs are competing for the same search intent.

= Features =
*   **Advanced Audit Engine:** Scans Posts, Pages, and Custom Post Types to find matching or similar slugs.
*   **Conflict Detection:** Categorizes conflicts into "Critical" (exact matches) and "Attention" (partial matches).
*   **Quick Resolution:** Fix issues directly from the dashboard using Canonical tags or 301 Redirects.
*   **Clean Interface:** Modern, localized dashboard integrated into your WordPress admin.
*   **Performance Focused:** Lightweight code that only runs when you start a scan.

== Installation ==

1. Upload the `seo-cannibalization-scout` folder to the `/wp-content/plugins/` directory.
2. Activate the plugin through the 'Plugins' menu in WordPress.
3. Access the audit dashboard via the "Cannibal Scout" menu in the sidebar.

== Frequently Asked Questions ==

= Does this plugin affect my site speed? =
No. The audit only runs manually when you click the "Start Scan" button. It does not add any scripts to your frontend.

= Can I choose which post types to audit? =
Yes, you can select any public post type (Posts, Pages, Recipes, etc.) before starting the scan.
